# 菜单栏图标显示模式技术方案

## 背景

现有菜单栏显示模式以主工具为中心,支持百分比、电池图标、电池 + 百分比、工具名 + 百分比。新增的 **图标** 与 **图标 + 数字** 模式不再只展示主工具,而是直接在菜单栏中展示 Claude 与 Codex 的品牌图标,用于并排观察两个工具的 5h 剩余额度。

## 显示模式

计划在 `MenuBarDisplayMode` 新增两个枚举值:

- `iconOnly`: 仅显示工具图标。
- `iconAndPercent`: 显示工具图标和对应的剩余百分比数字。

当显示模式为这两种之一时,设置页的「菜单栏显示」区域额外展示两个偏好项:

- 显示 Claude 图标
- 显示 Codex 图标

图标实际展示集合由「工具已启用」和「图标偏好已开启」共同决定。若筛选后没有可展示图标,菜单栏 label 回退到当前主工具快照,避免 `MenuBarExtra` 没有可点击锚点。

## 数据流

菜单栏 label 仍由 `MenuBarLabelView` 统一分发:

1. 读取 `settings.menuBarDisplayMode`。
2. 当模式为 `iconOnly` 或 `iconAndPercent` 时,根据 `settings` 中的 Claude / Codex 图标开关筛选工具。
3. 对每个可展示工具调用 `store.snapshot(for:)`,读取 `fiveHourRemainingPercent`。
4. 将 snapshot 传入图标视图,由图标视图负责把剩余百分比转成垂直填充区域。
5. `iconAndPercent` 模式在每个图标后追加 `UsageFormatters.percentText(...)`。

该模式不改变 `UsageStore.primarySnapshot` 的选择逻辑。原有主工具逻辑继续服务百分比、电池、工具名 + 百分比等单工具显示模式。

## 图标电量语义

图标模式复用 `UsageSnapshot.fiveHourRemainingPercent` 作为电量值:

- `100%` 表示图标内部完全填充。
- `0%` 或无数据表示图标内部不填充。
- 用量越多,剩余额度越低,白色填充区域从上往下减少,最终只保留底部填充或空图标。

图标内部只表达剩余额度,不再使用绿色、橙色、红色分级。告警分级仍可由图标旁的数字颜色表达。

## 绘制规则

`ToolUsageIcon` 从 asset catalog 中读取 `claude` / `codex` SVG,将图标 alpha 渲染为遮罩后生成菜单栏专用位图:

1. 使用 SVG alpha 生成原始图标遮罩。
2. 将遮罩按周围像素偏移绘制为半透明白色外边框。
3. 清空图标原始内部区域,使未填充区域保持透明。
4. 按剩余百分比裁剪底部高度,在遮罩内部绘制纯白填充。

因此图标在该模式下始终是白色填充,外边框透明度固定,不会随用量或告警级别改变。

## 文件改动范围

后续实现建议集中在这些文件:

- `agent-battery/Models/UsageModels.swift`: 扩展 `MenuBarDisplayMode`,并提供 `usesToolIcons` 之类的判断属性。
- `agent-battery/Stores/AppSettings.swift`: 增加 Claude / Codex 图标展示偏好的 `UserDefaults` 持久化。
- `agent-battery/Views/SettingsView.swift`: 在显示模式为图标类模式时展示两个图标开关。
- `agent-battery/Views/MenuBarLabelView.swift`: 为 `iconOnly` / `iconAndPercent` 添加菜单栏 label 分支。
- `agent-battery/Views/ToolUsageIcon.swift`: 新增品牌图标电量绘制视图。
- `l10n/displayMode.yaml` 与 `l10n/settings.yaml`: 增加显示模式与设置项文案。

## 设置持久化

计划新增 `UserDefaults` key:

- `showClaudeMenuBarIcon`,默认 `true`
- `showCodexMenuBarIcon`,默认 `true`

这两个 key 只影响图标类显示模式,不改变数据源启用状态。数据采集仍由 `claudeEnabled` 与 `codexEnabled` 控制。

## 边界情况

- 无数据或数据不可用: 图标内部填充高度为 `0%`,仅展示外边框。
- 工具被禁用: 不参与图标模式展示,除非后续产品决策明确要求禁用工具也显示空图标。
- 两个图标开关都关闭: 菜单栏 label 应保留一个回退内容,避免状态栏入口消失。
- 低电量告警: 图标本身不变色,仍绘制为白色;若模式为 `iconAndPercent`,数字可以继续沿用现有红/橙/默认色逻辑。
- 周用量: 图标模式只展示 5h 窗口剩余额度;周用量仍在下拉面板中展示。

## 验收点

- 显示模式列表包含「图标」与「图标 + 数字」。
- 选择任一图标类模式后,设置页出现 Claude / Codex 图标显示开关。
- 菜单栏可以同时显示 Claude 与 Codex 图标,也可以只显示其中一个。
- 图标外边框在不同剩余额度下保持一致。
- 图标内部填充随 5h 剩余额度从底部向上变化,剩余额度越低填充越少。
- 图标填充色始终为白色,不随 normal / warning / critical 状态变色。
