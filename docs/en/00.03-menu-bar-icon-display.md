# Menu Bar Icon Display Mode Technical Design

## Background

Existing menu bar display modes are primary-tool centered and support percent, battery icon, battery + percent, and tool name + percent. The new **icon** and **icon + number** modes no longer show only the primary tool; instead, they display Claude and Codex brand icons directly in the menu bar for side-by-side 5h remaining comparison.

## Display modes

Add two enum values to `MenuBarDisplayMode`:

- `iconOnly`: show tool icons only
- `iconAndPercent`: show tool icons plus remaining percentage text

When either icon mode is selected, settings should show two additional toggles in "Menu Bar Display":

- Show Claude icon
- Show Codex icon

Actual icon set is filtered by both "tool enabled" and "icon preference enabled". If no icon remains after filtering, menu bar label falls back to primary snapshot to keep a clickable `MenuBarExtra` anchor.

## Data flow

`MenuBarLabelView` remains the unified dispatcher:

1. Read `settings.menuBarDisplayMode`
2. For `iconOnly` or `iconAndPercent`, filter tools using Claude/Codex icon toggles in settings
3. For each visible tool, call `store.snapshot(for:)` and read `fiveHourRemainingPercent`
4. Pass snapshot to icon view; icon view converts remaining percentage into vertical fill area
5. In `iconAndPercent`, append `UsageFormatters.percentText(...)` after each icon

This mode does not alter `UsageStore.primarySnapshot` selection logic. Existing primary-tool flow still serves single-tool display modes.

## Icon battery semantics

Icon mode reuses `UsageSnapshot.fiveHourRemainingPercent` as battery value:

- `100%`: fully filled icon interior
- `0%` or no data: no interior fill
- More usage -> lower remaining quota -> white fill decreases from top to bottom

Icon interior only conveys remaining quota and no longer uses green/orange/red tiers. Alert tiers can still be represented by number color beside icon.

## Rendering rules

`ToolUsageIcon` reads `claude` / `codex` SVG assets and uses alpha mask rendering for menu-bar bitmaps:

1. Build base icon mask from SVG alpha
2. Draw a semi-transparent white outline around the mask via pixel offsets
3. Clear original interior region so unfilled region stays transparent
4. Crop by remaining percentage and draw pure white fill inside mask

So icons are always white-filled in this mode; outline opacity is fixed and does not vary by usage or alert level.

## File scope

Implementation should focus on:

- `agent-battery/Models/UsageModels.swift`: extend `MenuBarDisplayMode` and add helpers such as `usesToolIcons`
- `agent-battery/Stores/AppSettings.swift`: add UserDefaults persistence for Claude/Codex icon display preferences
- `agent-battery/Views/SettingsView.swift`: show icon toggles when icon display modes are active
- `agent-battery/Views/MenuBarLabelView.swift`: add label branches for `iconOnly` and `iconAndPercent`
- `agent-battery/Views/ToolUsageIcon.swift`: add brand icon battery rendering view
- `l10n/displayMode.yaml` and `l10n/settings.yaml`: add display-mode and settings copy

## Settings persistence

Planned new UserDefaults keys:

- `showClaudeMenuBarIcon` (default `true`)
- `showCodexMenuBarIcon` (default `true`)

These only affect icon display modes and do not change data-source enable state. Data collection is still controlled by `claudeEnabled` and `codexEnabled`.

## Edge cases

- No data/unavailable: icon interior fill = `0%`, show outline only
- Tool disabled: excluded from icon mode display
- Both icon toggles off: keep fallback content to avoid menu bar entry disappearance
- Low-quota alert: icon stays white; in `iconAndPercent`, number can keep existing red/orange/default logic
- Weekly usage: icon mode shows only 5h remaining; weekly stays in popover

## Acceptance points

- Display mode list includes "Icon" and "Icon + Number"
- Selecting icon mode reveals Claude/Codex icon toggles in settings
- Menu bar can show both icons or either single icon
- Outline remains consistent across different remaining values
- Interior fill changes from bottom to top with 5h remaining percentage
- Fill color always remains white regardless of normal/warning/critical level
