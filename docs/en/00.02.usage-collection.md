# Claude Code & Codex 5h/Weekly Usage Data Interface

This document describes how **Claude Code** and **OpenAI Codex CLI** expose local 5-hour and 7-day (weekly) quota usage rates. It does not prescribe a specific consumer implementation. Any tool can collect data through these two paths.

---

## Overview

The two agents do not share a common output channel, and their collection strategies differ:

| Agent | Where data appears | Collection method | Trigger | Installation needed |
|---|---|---|---|---|
| Claude Code | StatusLine hook stdin payload | Register a hook script | Every assistant turn status-line refresh | Yes |
| Codex CLI | rollout JSONL `event_msg.token_count.rate_limits` | Tail file | Every model `token_count` event | No |

Both channels are Anthropic/OpenAI implementation details, **not public APIs**. Fields may change between versions. Use lenient parsing with fallback defaults.

---

## 1. Claude Code

### 1.1 transcript JSONL does not carry quota

`assistant` lines in transcript only include per-turn token usage. Window-level quota percentages are only available in:

- Anthropic API streaming response (requires real request)
- **StatusLine hook payload** generated by Claude Code

Without remote API access, only the second path can be used.

### 1.2 StatusLine hook protocol

On each status-line refresh, Claude Code sends JSON to the executable configured in `~/.claude/settings.json` `statusLine.command` via **stdin** with 5-second timeout.

Key semantics:

- `rate_limits.five_hour.used_percentage`: 0-100 float
- `rate_limits.*.resets_at`: Unix seconds
- Hook script stdout is rendered as status-line text

### 1.3 Subscription-level impact

| Plan | `rate_limits` field |
|---|---|
| Pro / Max | Present with `five_hour` and `seven_day` |
| API key users | Missing |
| Free users | Missing |

When plan level changes, fields can appear/disappear suddenly. Collector must tolerate `rate_limits is None`.

### 1.4 settings.json registration

`statusLine.command` supports only one command globally. If multiple tools need payload access, use a wrapper script to fan out stdin.

### 1.5 Minimal hook template

The Chinese original includes a complete shell+Python hook template. Core implementation points:

- Read stdin in a loop with timeout to avoid hanging on partial payload
- Pipe stdin into Python process (instead of command argument) to avoid `ARG_MAX` issues on macOS
- Use atomic write (`tmp + rename`) to prevent partially written JSON
- Use proper file permissions (`chmod 0700`)

### 1.6 Stale recommendation

If file is not updated for a long time, it usually means no interaction or Claude Code exit. Recommended stale threshold: `(now - updated_at) > 10 min`.

## 2. Codex CLI

### 2.1 rollout JSONL

Codex writes token usage and quota into:

`~/.codex/sessions/YYYY/MM/DD/rollout-{timestamp}-{uuid}.jsonl`

No additional installation is needed; once Codex runs, files are available.

### 2.2 Key event: `token_count`

Important fields:

- `total_token_usage`: cumulative usage in current session
- `last_token_usage`: most recent turn
- `cached_input_tokens`: older versions may use `cache_read_input_tokens`, parser should accept both
- `rate_limits.primary` / `secondary`: see next section

### 2.3 Identify 5h / 7d slots

Do **not** hardcode by `primary`/`secondary` position. Map by `window_minutes`.

- `<= 300` -> 5-hour slot
- otherwise -> 7-day slot

### 2.4 Timestamp source

`updated_at` should use event `timestamp` instead of local `now()`, so replayed history keeps true ordering.

### 2.5 Session discovery (optional)

If you need to link quota to a running Codex process:

1. Use `ps` to find `codex` process (exclude unrelated processes)
2. Inspect open fds via `/proc/{pid}/fd/*` on Linux or `lsof -F pn -p{pid}` on macOS
3. Select file descriptors matching `rollout-*.jsonl`

If only account-level quota is needed, simply scan today's session directory and read most recently modified JSONL.

### 2.6 Persistence after exit

Rollout files remain after Codex exits but stop updating. Consumer tools should cache latest parsed values locally and fallback to cache when Codex is offline.

## 3. Unified `RateLimitInfo` model (reference)

The Chinese original provides a Rust struct that normalizes both sources into a single model with:

- `source`
- `five_hour_pct` + reset time
- `seven_day_pct` + reset time
- `updated_at`

Allow multiple entries for one source (for multi-config or multi-plan scenarios), and group by `source` in rendering.

## 4. Shared pitfalls

- Unstable schema: fields are implementation details, use lenient parsing
- PID reuse: verify process name still matches expected tool
- Missing quota fields are not always errors (free plan, API-key mode, timing issues)
- Cross-account contamination: multiple Codex accounts may share same local sessions directory
- Multiple Claude config dirs: `CLAUDE_CONFIG_DIR` can redirect config path
