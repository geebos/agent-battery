# macOS Menu Bar Usage Battery Tool MVP PRD

## 1. Product overview

Build a macOS menu bar utility that displays Claude Code / Codex remaining usage as a "battery percentage".

Core scenarios:

- Always show the primary tool's **remaining 5h usage percentage** in the menu bar
- Click menu bar item to view Claude Code / Codex 5h usage, weekly usage, remaining quota, and reset time
- Help users quickly judge whether they are close to quota exhaustion and need to switch tools or control usage pace

One-line product statement:

> A macOS menu bar usage battery meter so developers always know how much Claude Code / Codex quota is left.

## 2. Target users

Frequent Claude Code / Codex users, especially:

- Individual developers who use AI coding agents for long sessions
- Developers using both Claude Code and Codex
- Heavy users sensitive to 5-hour and weekly limits

Users need quick answers to:

- Is the current 5-hour remaining quota about to run out?
- Is this week's remaining quota too low?
- Should I switch tools or reduce usage pace?

## 3. User value

- **Risk visibility at a glance**: see quota risk without opening a web page or terminal
- **Low interruption**: persistent yet compact menu bar presence
- **Fast detail lookup**: see 5h and weekly status within one second after click
- **Dual-tool comparison**: monitor Claude Code and Codex side by side

## 4. MVP scope

### 4.1 In scope

- Persistent macOS menu bar item
- Show remaining 5h usage percentage for primary tool
- Click to open Claude Code / Codex detail popover
- Display remaining percentage and reset time for 5h + weekly usage
- Support both Claude Code and Codex data sources
- Auto refresh every 1-5 minutes
- Launch at login
- Basic settings

### 4.2 Out of scope

- Complex historical charts
- Multi-account support
- Team usage
- Mobile app
- Cloud sync
- Auto-purchase or auto-expand quota
- Complex notification strategies
- Manual refresh

## 5. Core features

### 5.1 Menu bar display

Default: show remaining 5h usage percentage for the primary tool.

Examples:

```text
🔋 62%
```

Compact mode:

```text
62%
```

Display options:

- Percentage only: `62%`
- Battery icon + percentage: `🔋 62%`
- Tool name + percentage: `Claude 62%`

Status color suggestions:


| Remaining | State    | UI                           |
| --------- | -------- | ---------------------------- |
| 41-100%   | Normal   | default color                |
| 16-40%    | Warning  | yellow/system warning color  |
| 0-15%     | Critical | red/system destructive color |


Semantic conventions:

- UI shows "remaining quota percentage"
- 100% means sufficient quota, 0% means exhausted
- Lower percentage means heavier usage and closer to exhaustion
- If source provides `used_percentage`, convert before display: `remaining_percentage = clamp(100 - used_percentage, 0, 100)`

When data is unavailable:

- Show `--%`
- Explain reason in popover
- Wait for next automatic refresh (no manual refresh action)

### 5.2 Detail popover

Click menu bar icon to open details.

The popover includes:

- Tool name: Claude Code / Codex
- Remaining 5h percentage
- 5h reset time or time remaining
- Remaining weekly percentage
- Weekly reset time or time remaining
- Last updated time
- Settings entry
- Quit entry

### 5.3 Data sources

MVP is based on local collection. See [00.02.usage-collection.md](./00.02.usage-collection.md).

Claude Code:

- Prefer local JSON written by StatusLine hook
- Support 5h + weekly usage
- Convert source `used_percentage` to UI remaining percentage
- Show unavailable state when `rate_limits` is missing
- Explicit stale-state messaging

Codex:

- Prefer `event_msg.token_count.rate_limits` in rollout JSONL
- Support 5h + weekly usage
- Convert source usage values to remaining percentage for UI
- Tolerate missing data, schema changes, or stale values

Unified shape:

```json
{
  "tool": "claude_code",
  "fiveHourRemainingPercent": 62,
  "weeklyRemainingPercent": 31,
  "fiveHourResetAt": "2026-04-29T18:30:00Z",
  "weeklyResetAt": "2026-05-04T00:00:00Z",
  "updatedAt": "2026-04-29T17:06:00Z",
  "status": "available"
}
```

Data states:


| State         | Meaning                       | UI                                            |
| ------------- | ----------------------------- | --------------------------------------------- |
| `available`   | Data available                | Show remaining % and reset time               |
| `unavailable` | Source has no usage data now  | Show `--%` and reason                         |
| `stale`       | Data exceeds freshness window | Show previous value + "updated N minutes ago" |
| `error`       | Read/parse failed             | Show error summary + auto-retry status        |


### 5.4 Refresh behavior

- Auto-refresh enabled, default 1 minute
- Configurable options: 1 / 3 / 5 minutes
- Keep previous data on refresh failure, but mark status
- Trigger a lightweight refresh when popover opens
- No manual refresh button or shortcut

### 5.5 Settings page

Settings items:

- Enabled tools: Claude Code / Codex
- Primary display tool: Claude Code / Codex / auto choose lowest remaining quota
- Menu bar style: percent only / battery + percent / tool name + percent
- Refresh interval: 1 / 3 / 5 minutes
- Alert thresholds: warning and critical
- Launch at login
- Show weekly usage
- Data file path configuration

## 7. Key UX requirements

- Menu bar label must stay compact (default <= 8 chars)
- Detail popover visible within one second after click
- Near-exhaustion quota should have clear but restrained visual cue
- Unavailable data must explain reason to avoid confusion with 0%
- No mandatory token or API key input
- No dependency on web login state
- Background refresh should not noticeably increase CPU or battery usage

## 8. Technical recommendations

Suggested stack:

- Swift / SwiftUI
- macOS `MenuBarExtra`
- `UserDefaults` for regular settings
- `ServiceManagement` for Launch at Login
- Background `Timer` or structured concurrency refresh
- Keychain reserved for future token storage needs

Data layer abstraction:

```text
UsageProvider
├── ClaudeCodeUsageProvider
└── CodexUsageProvider
```

Core model:

```swift
struct UsageSnapshot {
    let tool: UsageTool
    let fiveHourRemainingPercent: Double?
    let weeklyRemainingPercent: Double?
    let fiveHourResetAt: Date?
    let weeklyResetAt: Date?
    let updatedAt: Date?
    let status: UsageStatus
    let message: String?
}
```

Implementation principles:

- Providers only collect/parse; no UI knowledge
- UI consumes unified `UsageSnapshot`
- Parse external data leniently to avoid crashes on field changes
- Avoid main-thread blocking when reading local files
- Output stale/error states consistently in data layer

## 9. Acceptance criteria

### 9.1 Menu bar

- Usage label appears after app launch
- Show remaining 5h percentage when data exists
- Show `--%` when data unavailable
- Style/color changes when remaining percentage crosses thresholds

### 9.2 Popover

- Click menu bar item opens details
- Show Claude Code / Codex 5h + weekly remaining percentages
- Show reset time or time remaining
- Show last updated or stale status
- Settings / Quit entries are clickable

### 9.3 Data

- Claude provider reads local usage file
- Codex provider reads rollout JSONL usage events
- App does not crash for missing fields, missing files, or parse failures
- Stale data clearly indicated

### 9.4 Settings

- Enable/disable Claude Code and Codex
- Switch menu bar display mode
- Set refresh interval
- Set warning and critical thresholds
- Enable/disable launch at login
