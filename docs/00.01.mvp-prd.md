# Mac 状态栏 Usage 电量工具 MVP PRD

## 1. 产品概述

做一个 macOS 状态栏工具，用“电量百分比”的形式展示用户 Claude Code / Codex 的 usage 剩余额度。

核心场景：

- 状态栏常驻显示当前主工具的 **5h usage 剩余百分比**
- 用户点击状态栏后展开详情，查看 Claude Code / Codex 的 5h usage、weekly usage、剩余额度或重置时间
- 用户可以快速判断是否接近额度耗尽，是否需要切换工具或控制使用节奏

产品一句话：

> 一个 macOS 状态栏 usage 电量表，让开发者随时知道 Claude Code / Codex 还剩多少可用额度。

## 2. 目标用户

面向高频使用 Claude Code、Codex 的开发者，尤其是：

- 长时间使用 AI coding agent 的个人开发者
- 同时使用 Claude Code 与 Codex 的开发者
- 对 5 小时窗口与 weekly usage 较敏感的重度用户

用户需要快速判断：

- 当前 5 小时窗口剩余额度是否快耗尽
- 本周剩余额度是否偏低
- 是否需要切换工具或控制使用节奏

## 3. 用户价值

- **一眼判断风险**：无需打开网页或命令行即可看到 usage 是否危险
- **降低中断感**：状态栏常驻但占用空间小，不打扰工作流
- **快速查看详情**：点击后 1 秒内看到 5h 与 weekly usage
- **支持双工具对比**：同时查看 Claude Code / Codex 的当前额度状态

## 4. MVP 范围

### 4.1 首版要做

- macOS 状态栏常驻
- 状态栏显示当前主工具的 5h usage 剩余百分比
- 点击状态栏显示 Claude Code / Codex 详情弹窗
- 展示 5h usage、weekly usage 的剩余百分比和重置时间
- 支持 Claude Code / Codex 两个数据来源
- 支持自动每 1-5 分钟刷新
- 支持开机启动
- 支持基础设置项

### 4.2 首版不做

- 复杂历史图表
- 多账号
- 团队 usage
- 手机端
- 云同步
- 自动购买或扩容额度
- 复杂通知策略
- 手动刷新

## 5. 核心功能

### 5.1 状态栏展示

状态栏默认显示当前主工具的 5h usage 剩余百分比。

示例：

```text
🔋 62%
```

紧凑模式：

```text
62%
```

显示模式：

- 只显示百分比：`62%`
- 显示电量图标 + 百分比：`🔋 62%`
- 显示工具名 + 百分比：`Claude 62%`

状态栏颜色建议：

| 剩余百分比 | 状态 | 表现 |
|---|---|---|
| 41-100% | 正常 | 默认颜色 |
| 16-40% | 提醒 | 黄色或系统 warning 色 |
| 0-15% | 警告 | 红色或系统 destructive 色 |

语义约定：

- UI 展示的是“剩余额度百分比”
- 100% 代表额度充足，0% 代表额度耗尽
- 百分比越低，代表用得越多、越接近耗尽
- 如果底层数据源提供的是 `used_percentage`，展示前需要转换为 `remaining_percentage = clamp(100 - used_percentage, 0, 100)`

当数据不可用时：

- 显示 `--%`
- 点击弹窗中展示不可用原因
- 等待下一次自动刷新，不提供手动刷新入口

### 5.2 点击弹窗

点击状态栏图标后显示详情。

示例：

```text

██████░░░░ 62%
Resets in 1h 24m

Weekly Remaining
███░░░░░░░ 31%
Resets Monday 00:00

Codex

5h Remaining
██░░░░░░░░ 18%
Resets in 3h 12m

Weekly Remaining
████░░░░░░ 44%
Resets Monday 00:00

```

弹窗信息：

- 当前工具名称：Claude Code / Codex
- 5h usage 剩余百分比
- 5h usage 重置时间或剩余时间
- weekly usage 剩余百分比
- weekly usage 重置时间或剩余时间
- 最近更新时间
- 设置入口
- 退出应用入口

### 5.3 数据来源

MVP 数据来源以本地采集为主，具体采集方案参考 [00.02.usage-collection.md](./00.02.usage-collection.md)。

Claude Code：

- 优先读取 StatusLine hook 写出的本地 JSON 文件
- 支持 5h usage 与 weekly usage
- 将底层 `used_percentage` 转换为 UI 展示用的剩余百分比
- 当 `rate_limits` 缺失时展示数据不可用
- 对 stale 数据做明确提示

Codex：

- 优先读取 rollout JSONL 中的 `event_msg.token_count.rate_limits`
- 支持 5h usage 与 weekly usage
- 将底层使用率转换为 UI 展示用的剩余百分比
- 对缺失、格式变化或 stale 数据做容错

统一数据结构：

```json
{
  "tool": "claude_code",
  "fiveHourRemainingPercent": 62,
  "weeklyRemainingPercent": 31,
  "fiveHourResetAt": "2026-04-29T18:30:00Z",
  "weeklyResetAt": "2026-05-04T00:00:00Z",
  "updatedAt": "2026-04-29T17:06:00Z",
  "status": "available"
}
```

数据状态：

| 状态 | 含义 | UI 表现 |
|---|---|---|
| `available` | 数据可用 | 正常展示剩余百分比和重置时间 |
| `unavailable` | 当前工具未提供 usage | 展示 `--%` 和原因 |
| `stale` | 数据超过有效时间未更新 | 展示旧值和“更新于 N 分钟前” |
| `error` | 读取或解析失败 | 展示错误摘要和自动重试状态 |

### 5.4 刷新机制

- 支持自动刷新，默认 1 分钟
- 设置页可选刷新频率：1 分钟 / 3 分钟 / 5 分钟
- 刷新失败不清空旧数据，但需要标记状态
- 弹窗打开时可触发一次轻量刷新
- 不提供手动刷新按钮或快捷操作

### 5.5 设置页

设置项：

- 启用工具：Claude Code / Codex
- 主显示工具：Claude Code / Codex / 自动选择剩余额度最低的工具
- 状态栏显示内容：只显示百分比 / 电量图标 + 百分比 / 工具名 + 百分比
- 刷新频率：1 分钟 / 3 分钟 / 5 分钟
- 告警阈值：提醒阈值、警告阈值
- 是否开机启动
- 是否显示 weekly usage
- 数据文件路径配置

## 7. 关键体验要求

- 状态栏内容必须足够短，默认不超过 8 个字符
- 点击状态栏后 1 秒内看到详情
- usage 剩余额度接近耗尽时有明显但克制的视觉提醒
- 数据不可用时说明原因，不让用户误以为剩余额度为 0
- 不强制用户输入 token 或 API key
- 不依赖网页登录态
- 后台刷新不能明显增加 CPU 或电量消耗

## 8. 技术方案建议

推荐技术栈：

- Swift / SwiftUI
- macOS `MenuBarExtra`
- `UserDefaults` 存储普通设置
- `ServiceManagement` 支持 Launch at Login
- 后台 `Timer` 或 structured concurrency 定时刷新
- Keychain 预留 token 存储能力

数据层抽象：

```text
UsageProvider
├── ClaudeCodeUsageProvider
└── CodexUsageProvider
```

核心模型：

```swift
struct UsageSnapshot {
    let tool: UsageTool
    let fiveHourRemainingPercent: Double?
    let weeklyRemainingPercent: Double?
    let fiveHourResetAt: Date?
    let weeklyResetAt: Date?
    let updatedAt: Date?
    let status: UsageStatus
    let message: String?
}
```

实现原则：

- provider 只负责采集和解析，不关心 UI
- UI 只消费统一的 `UsageSnapshot`
- 所有外部数据解析都要宽松处理，避免字段变化导致崩溃
- 读取本地文件时避免阻塞主线程
- stale 判定与错误状态在数据层统一输出

## 9. 验收标准

### 9.1 状态栏

- 应用启动后状态栏出现 usage 显示
- 有可用数据时显示 5h usage 剩余百分比
- 数据不可用时显示 `--%`
- usage 剩余百分比低于阈值时状态栏颜色或样式变化

### 9.2 弹窗

- 点击状态栏可以打开详情弹窗
- 弹窗展示 Claude Code / Codex 的 5h 与 weekly 剩余百分比
- 弹窗展示 reset 时间或剩余时间
- 弹窗展示最近更新时间或 stale 状态
- Settings / Quit 可点击

### 9.3 数据

- Claude Code provider 可以读取本地 usage 文件
- Codex provider 可以读取本地 rollout JSONL usage 信息
- 字段缺失、文件不存在、JSON 解析失败时应用不崩溃
- stale 数据有明确提示

### 9.4 设置

- 可以启用或禁用 Claude Code / Codex
- 可以切换状态栏显示模式
- 可以设置刷新频率
- 可以设置提醒和警告阈值
- 可以开启或关闭开机启动
